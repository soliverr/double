## Генеральная идея

Наиглавнейшая идея, парадигма, данного проекта - создание персонально личной цифровой копии знаний, умений и навыков для взаимодействия в цифровом мире, вычислительной экосистеме.

Личная цифровая копия первоначально является набором навыков взаимодействия с информационными системами в самом широком смысле этого слова, взаимодействия посредством вычислительных устройств (компьютеров, смартфонов, гаджетов) и взаимодействия, направленного на вас со стороны цифрового мира.

Цифровая копия изначально создаётся как персональная копия ваших навыков, которая может и должна непрерывно эволюционировать и улучшаться.

По желанию и согласию участников проекта, локальные персональные цифровые копии могут объединяться для создания общего, универсального и провалидированного всеми участниками проекта Double общедоступного агента, объединяющего лучшие персональные навыки участников.

Универсальный агент Double может выступать в качестве гуру для локальных цифровых копий новых участников или пользователей проекта Double.

Таким образом, здесь заявляется основополагающий принцип данного проекта:

 **Создание открытой системы накопления и взаимообмена знаниями между всеми участниками проекта Double.**
 
*Сами накапливаемые знания и навыки могут быть как **открытыми** и доступными любому участнику проекта и внешнему пользователю, так и **закрытыми**, доступными индивидуально или по платной подписке.*

## Цели проекта

Значимые для всех участников цели проекта содержаться в каталоге [goals](../../goals/goals.md).

Для уточнения и планирования задач по достижению целей, в каталоге [goals](../../goals/goals.md) создаются каталоги целей, под целей с файлами, содержащими списки решаемых задач.

Для сложных целей, описываемых несколькими словами, необходимо использовать [Camel case](https://en.wikipedia.org/wiki/Camel_case) именование.

Цель - это описание того, что необходимо достичь.  Для того, что бы получить автоматизированный способ достижения цели, необходимо договориться о формальном описании цели.

Цель должна быть описана таким образом, что бы можно было как можно чётче получить параметры, определяющие достижение цели.

Для каждого аспекта достижения цели должны быть описаны параметры валидации.

Достижение цели представляется как план, выполнение этапов которого приближает к достижению. Необходимо, чтобы  план достижения целей был ациклическим направленным графом, переводящим исполнителя из одного состояние в другое.

Каждое состояние - это набор достижений, которые приобретаются по мере выполнения плана. Так как в разных окружениях достижения могут уже существовать, то в план может быть несколько точек входа.

План может быть большим и сложным, части плана могут использоваться для других более мелких целей. Поэтому из плана может быть несколько точек выхода.

Точки входа и выхода - это состояния, полноту которых необходимо оценить.

Движение по плану - это получение достижений, как только все достижения для какого-то состояние активны, включается процедура выбора дальнейшего направления для перехода в следующее состояние.

План не обязательно может иметь один путь из одного состояния в другое и к конечной цели. Пути плана должны конкурировать между собой и участвовать в механизме естественного отбора.

Участники проекта Double могут по желанию делиться планами с агентом Double.

Должна быть предусмотрена процедура слияния планов, сколько версий планов поддерживать в актуальном состоянии.

## Умение

Реализация агентом Double плана достижения цели формирует умение данного агента. В процессе формирования умения происходит накопление знаний, формирование и закрепление навыков. 

Может существовать одновременно несколько похожих умений, которые могут быть использованы для реализации плана.

Умение - это точный алгоритм, который воспроизведён данным агентом, в процессе воспроизведения которого достигаются состояния и рассчитываются операционные метрики для данного плана.

Могут быть циклы обратной связи (рефлексии), во время которых происходит анализ хода выполнения плана и уточняются критерии эффективности его исполнения.

Критерии эффективности индивидуальны и зависимы от времени и среды исполнения.

Таким образом, изменяя критерии оценки эффективности, могут быть выбраны разные планы, которые существуют одновременно в составе умения.

Агент Double должен применять набор таких однотипных умений при каждой реализации плана.

## Способности

Способности - это то что способствует формированию умений и приобретению навыков.

В вычислительных системах способностями являются:
* аппаратные средства
	* ноутбук
	* смартфон
* операционная система
* оболочка Shell
* язык написания программ
* *что ещё?*

Основные признаки способности: то что помогает создавать умения и сворачивать  их в автоматические навыки. Помогает достижению многих целей и формированию разнообразных навыков.

У способности также должны быть метрики:
* сколько умений и навыков помогла сформировать
* время, затраченное на формирование
* сложность планов для достижения целей, в которых пригодилась данная способность

Способность может быть неотъемлемой, как аппаратные средства, так и подгружаемой/дополняемой - выбор другого Shell или языковой среды для формирования умений и навыков.

## Умения

Умение - это реализованный план, который привёл к цели. То есть, пройдены все этапы плана, получены их метрики, получена цель, рефлексия - анализ, что план привёл именно к той цели, которую хотели достичь.

Цель агента - автоматизировать умения и сворачивать их в навыки.

## Навыки

Общезначимые навыки размещаются в каталоге [skills](../../skills/skills.md).

Для каждого выделенного навыка создаётся отдельный каталог. Для наименования навыков используются общеизвестные слова английского языка. 
Для сложных навыков, описываемых несколькими словами, необходимо использовать [Camel case](https://en.wikipedia.org/wiki/Camel_case) именование.

Навык - это автоматическое действие, которое может быть применено к какому-то этапу плана для достижения цели при решении конкретной задачи. То есть, какая-то функция или программа: есть входные параметры, есть выходной результат

Должны быть метрики:
* стоимость
* затраты времени
* верификация, что получен ожидаемый результат: тип результата, смысл результата.

Навыки - это доведённые до автоматизма умения. При реализации планов достижения целей и в процессе последующего анализа, принимается решение о сворачивании части умений или умения целиком в более сложный навых.

Навык - это процедура или программа с чёткими входными и чёткими выходными параметрами, которая обязательно имеет метрики и валидацию полученного результата.
# Агент Double

 Агент создаётся каждый раз индивидуально участником проекта для выбранной информационной среды: личный компьютер, ноутбук, смартфон, облачный сервис.

Агент - не монолитная программа, а множество взаимодействующих умений и навыков ("микросервисы"). Должен быть супервизор, для координации взаимодействия.

Супервизор - и есть тот общий/универсальный агент, который разрабатывается участниками проекта.

Можно выгружать и загружать "личности" других агентов: цели, умения, навыки, знания.

Должна быть процедура репликации нового персонального агента, которую запускает супервизор. Значит может быть сразу несколько "личностей" в одной среде обитания.

Экземпляр агента  - это информационный геном личности, который может быть воспроизведён множество раз. Необходим закон естественного отбора определяющий правила выживания экземпляров агента.
